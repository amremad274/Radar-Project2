Pulse-Doppler Linear Frequency Modulated (LFM) Radar System

Technical Validation & Academic Showcase

Project Objective

This project demonstrates a Pulse-Doppler LFM Radar system that can measure both how far away (range) and how fast (velocity) a target is moving. Think of it like this:

  • LFM Chirp: We send out a signal that changes frequency over time (like a police siren going up in pitch)
  • Matched Filter: We compress the received signal to find exactly when it came back (tells us range)
  • Doppler Processing: We analyze how the frequency changed due to motion (tells us velocity)
  • 2D Map: We combine both to create a complete picture showing all targets with their range and speed

🔑 Key Takeaway

The system works across 13 blocks in 3 layers:

  1. Signal Layer (Blocks 1–5): Generate, transmit, propagate, add noise, reshape
  2. Processing Layer (Blocks 6–10): Matched filter, range estimation, Doppler FFT, interpolation, RD map
  3. Analysis Layer (Blocks 11–13): SNR tracking, Detection (Pd/Pfa), Latency measurement

By combining all layers, we get a complete picture: where targets are, how fast they move, how reliably we detect them, and how fast the system runs!

Detailed Block Explanations

Here's what happens at each processing stage. let's explain each step.

Block 1

LFM Signal Generation

� What It Does

Creates a chirp signal that sweeps from low frequency to high frequency over time. Think of it like a whistle that goes from low pitch to high pitch smoothly.

📐 Key Equations

  • Pulse Duration: $T = \frac{N}{f_s} = 25.6$ μs
  • Chirp Rate: $k = \frac{B}{T} = 1.95 \times 10^{11}$ Hz/s (how fast frequency changes)
  • Signal: $s(t) = e^{j\pi kt^2}$ (phase grows with time squared)

🔢 MATLAB Code

T = pulse_len/fs;           % Calculate duration
k = B / T;                  % Calculate chirp rate
pulse = exp(1j*pi*k*t_pulse.^2);  % Generate LFM signal

� Why It Matters

The chirp can be compressed later by the matched filter, turning a long 25.6 μs pulse into a sharp peak. This gives us good range resolution (30 m) while maintaining enough energy for detection.

Left: LFM chirp real part — Right: Instantaneous frequency sweep 0→5 MHz

📊 Code Tracing Values

Input: None (generation from parameters)

Parameters Used: pulse_len = 512, fs = 20×10⁶ Hz, B = 5×10⁶ Hz

Computed: T = 25.6 μs, k = 1.953×10¹¹ Hz/s

Output: pulse ∈ ℂ⁵¹² (512 complex samples)

Block 2

Pulse Train Formation

� What It Does

Takes the single chirp pulse and repeats it 128 times with gaps in between. Like sending out 128 radar "pings" in a row.

📐 Key Equations

  • PRI (Pulse Repetition Interval): Time between pulses = 35.6 μs
  • PRF (Pulse Repetition Frequency): $\text{PRF} = \frac{1}{\text{PRI}} = 28.09$ kHz

� MATLAB Code

for i = 1:num_pulses
    tx = [tx pulse zeros(1, pulse_gap)];  % Add pulse + gap
end

💡 Why It Matters

Multiple pulses let us measure velocity using the Doppler effect. Each pulse samples the target at a different time, and we can see how the phase changes from pulse to pulse (this tells us velocity). More pulses = better velocity accuracy.

First 4 pulses of the pulse train — shaded gaps show the 200-sample silent intervals

📊 Code Tracing Values

Input: Single pulse ∈ ℂ⁵¹²

Parameters Used: num_pulses = 128, pulse_gap = 200 samples

Computed: pulse_total_len = 712 samples (512 + 200)

Output: tx ∈ ℂ⁹¹'¹³⁶ (128×712 = 91,136 samples)

Block 3

Channel Propagation & Doppler Shift

� What It Does

Simulates what happens when the signal travels to the target and back. Two things happen: (1) the signal gets delayed (tells us range), and (2) the frequency gets shifted (tells us velocity).

📐 Key Equations

  • Delay Time: $\tau = \frac{2R}{c} = 3.33$ μs (signal travels 500m + 500m back)
  • Doppler Shift: $f_d = \frac{2v}{\lambda} = 3200$ Hz (moving target changes frequency)
  • Received Signal: Delayed copy with frequency shift applied

� MATLAB Code

tau = 2 * target_range / c;        % Calculate delay
rx = [zeros(1, delay_samples) tx]; % Delay signal

fd = 2 * target_velocity / lambda; % Calculate Doppler
rx = rx .* exp(1j*2*pi*fd*t);      % Apply Doppler shift

� Why It Matters

This is the physics of radar! The delay tells us how far (500m → 3.33 μs delay), and the Doppler tells us how fast (200 m/s → 3200 Hz shift). The rest of the system is about extracting these two pieces of information from the noisy received signal.

📊 Code Tracing Values

Input: tx ∈ ℂ⁹¹'¹³⁶

Computed: tau = 3.33 μs, delay_samples = 67, fd = 3200 Hz

Output: rx ∈ ℂ⁹¹'²⁰³ (91,136 + 67 = 91,203 samples, Doppler modulated)

Block 4

Additive White Gaussian Noise (AWGN)

� What It Does

Adds random noise to the signal to make it realistic. Real radar always has noise from electronics, atmosphere, etc.

📐 Key Equations

  • Noise Model: $n(t) \sim \mathcal{CN}(0, \sigma^2)$ (complex Gaussian noise)
  • Noise Power: $\sigma^2 = 0.01$ → SNR ≈ 20 dB before processing

� MATLAB Code

noise_power = 0.01;
noise = sqrt(noise_power/2) * (randn(size(rx)) + 1j*randn(size(rx)));
rx = rx + noise;  % Add noise to received signal

� Why It Matters

Testing with noise proves the system works in realistic conditions. The matched filter will later improve SNR from 20 dB to ~44 dB (24 dB gain). This shows our processing is robust!

Block 5

Slow-Time / Fast-Time Matrix Formation

� What It Does

Reorganizes the long 1D signal into a 2D matrix (table). Rows = range bins, Columns = pulse numbers. This makes it easy to process range and Doppler separately.

📐 Key Concept

  • Fast-Time (down): Each row is a range bin (like 0m, 7.5m, 15m, ...)
  • Slow-Time (across): Each column is a pulse number (1, 2, 3, ..., 128)
  • Matrix Size: 712 rows × 128 columns

🔢 MATLAB Code

rx_matrix = reshape(rx, pulse_total_len, num_pulses);
% Now: rows = range bins, columns = pulses

💡 Why It Matters

Think of it like a spreadsheet: look down a column to see one pulse at all ranges, or look across a row to see one range across all pulses. This makes it easy to process range (down columns) and Doppler (across rows) separately.

Block 6

Matched Filter (Pulse Compression)

� What It Does

This is the magic step! Compresses the long 25.6 μs chirp into a sharp peak. Like squeezing a sponge - the energy concentrates into a narrow pulse, making it easy to see where the target is.

📐 Key Equations

  • Filter: $h(t) = s^*(-t)$ (time-reversed conjugate of transmitted pulse)
  • Operation: Convolution (cross-correlation) with each received pulse
  • Gain: Processing gain = 24.1 dB (SNR improves from 20 dB to 44 dB!)

� MATLAB Code

tx_mf = conj(fliplr(pulse));  % Create matched filter
for i = 1:num_pulses
    corr = conv(rx_matrix(:,i), tx_mf, 'full');  % Compress
    range_matrix(:,i) = corr;  % Store result
end

� Why It Matters

Key benefit: Turns a long pulse (bad resolution) into a short peak (good resolution). The 5 MHz bandwidth gives us 30m range resolution. Also adds 24 dB processing gain - makes the target much easier to detect in noise!

Block 7

Range Estimation

� What It Does

Finds which range bin has the strongest signal. That's where the target is! Then converts the bin number to actual distance in meters.

📐 Key Equations

  • Range Formula: $R = n \cdot \frac{c}{2f_s}$ where $n$ is bin number
  • Bin Spacing: Each bin = 7.5 m
  • Method: Sum across all 128 pulses, find the peak

� MATLAB Code

[~, range_bin] = max(sum(abs(range_matrix), 2));  % Find peak bin
range_axis = (0:pulse_total_len-1) * (c/(2*fs));   % Convert to meters
range_est = range_axis(range_bin);                 % Get range estimate

� Why It Matters

This gives us the final range answer! For our target at 500m, we find the peak around bin 67. Summing across all 128 pulses adds another 21 dB of SNR improvement (on top of the 24 dB from matched filter).

Block 8

Doppler Processing (Slow-Time FFT)

� What It Does

Now we know the range, extract that range bin and look at how it changes across the 128 pulses. FFT converts this into frequency domain to find the Doppler shift.

📐 Key Equations

  • Doppler FFT: 1024-point FFT across 128 pulses
  • Velocity from Doppler: $v = \frac{f_d \lambda}{2}$
  • Frequency Resolution: 27.4 Hz per bin

� MATLAB Code

slow_time_signal = range_matrix(range_bin, :);
slow_time_signal = detrend(slow_time_signal);          % Remove DC
slow_time_signal = slow_time_signal .* hamming(128).'; % Window

doppler_fft = fftshift(fft(slow_time_signal, 1024));   % FFT
f_axis = (-512:511)*(PRF/1024);                        % Frequency axis

� Why It Matters

The FFT finds the Doppler frequency (3200 Hz), which we convert to velocity (200 m/s) using $v = f_d\lambda/2$. Detrend removes stationary clutter, Hamming reduces sidelobes. The peak in the spectrum tells us how fast the target is moving!

Block 9

Quadratic Peak Interpolation

� What It Does

FFT bins are discrete, but the true peak might be between bins. This uses the peak and its neighbors to estimate the exact frequency with sub-bin accuracy. Like refining your measurement!

📐 Key Equations

  • Parabolic Fit: $\delta = \frac{0.5(\alpha - \gamma)}{\alpha - 2\beta + \gamma}$ (fits parabola through 3 points)
  • Refined Frequency: $\hat{f}_d = f[k] + \delta \cdot \Delta f$
  • Accuracy: Improves from ±0.11 m/s to ±0.01 m/s

� MATLAB Code

alpha = abs(doppler_fft(idx_fd-1));  % Left neighbor
beta  = abs(doppler_fft(idx_fd));    % Peak
gamma = abs(doppler_fft(idx_fd+1));  % Right neighbor

delta = 0.5 * (alpha-gamma)/(alpha-2*beta+gamma);  % Offset
fd_est = f_axis(idx_fd) + delta*(PRF/N_fft);       % Refined freq
velocity_est = (fd_est * lambda)/2;                 % Final velocity

💡 Why It Matters

Without interpolation, we're limited to bin spacing accuracy. With it, we get 10x better precision! This is important for accurate velocity measurement, especially for slow-moving targets.

Block 10

2D Range-Doppler Map Generation

� What It Does

Creates the final 2D image showing ALL targets at ALL ranges and ALL velocities simultaneously. This is the complete radar picture!

📐 Key Equations

  • 2D Processing: FFT along slow-time (Doppler) for each range bin
  • Output Size: 712 range bins × 1024 Doppler bins
  • Display: $P_{\text{dB}} = 20\log_{10}(|S|)$ (convert to dB for visualization)

� MATLAB Code

range_window = hamming(pulse_total_len);
doppler_window = hamming(num_pulses);
range_matrix_win = range_matrix .* (range_window * doppler_window.');

RD_map = fftshift(fft(range_matrix_win, N_fft, 2), 2);
RD_map_abs = abs(RD_map);
RD_map_abs = RD_map_abs / max(RD_map_abs(:));

velocity_axis = (f_axis * lambda)/2;

imagesc(range_axis, velocity_axis, 20*log10(RD_map_abs.'));
xlabel('Range (m)'); ylabel('Velocity (m/s)');

� Why It Matters

This is the final output! A 2D image where the X-axis is range (0-5000m) and Y-axis is velocity (-1750 to +1750 m/s). Our target appears as a bright spot at (500m, 200 m/s). In a real radar, multiple targets would show up as multiple bright spots. The map shows everything at once - no need to process range and velocity separately!

Block 11

SNR Analysis

🔍 What It Does

Tracks the Signal-to-Noise Ratio at every stage of the processing pipeline. The SNR starts low (20 dB) and improves dramatically through two stages of processing gain.

📐 Key Equations

  • Input SNR (Eq.19): $\text{SNR}_{\text{in}} = \frac{P_s}{\sigma^2} = 20$ dB
  • After Matched Filter (Eq.20): $\text{SNR}_{\text{MF}} = \text{SNR}_{\text{in}} \cdot BT = 20 + 24.1 = 44.1$ dB
  • After Coherent Integration (Eq.21): $\text{SNR}_{\text{CI}} = \text{SNR}_{\text{MF}} \cdot M = 44.1 + 21 = 65.1$ dB

🔢 MATLAB Code

signal_power_in = mean(abs(pulse).^2);
SNR_in_dB  = 10*log10(signal_power_in / noise_power);
G_MF_dB    = 10*log10(B*T);         % ~24.1 dB
SNR_MF_dB  = SNR_in_dB + G_MF_dB;
G_CI_dB    = 10*log10(num_pulses);  % ~21 dB
SNR_CI_dB  = SNR_MF_dB + G_CI_dB;
SNR improvement across processing stages — total gain of 42.14 dB

💡 Why It Matters

SNR improvement across processing stages — total gain of 42.14 dB

The total processing gain is 45 dB (factor of ~31,623×). The matched filter exploits the known waveform shape; coherent integration exploits phase consistency across pulses while noise averages out.

📊 Code Tracing Values

Input SNR: 20 dB (noise_power = 0.01)

After Matched Filter: 44.1 dB (+24.1 dB, BT = 256)

After Coherent Integration: 65.1 dB (+21 dB, M = 128)

Total Gain: 45.1 dB

Block 12

Detection Theory — Pd & Pfa

🔍 What It Does

Determines how reliably the radar detects the target. Uses the Neyman-Pearson criterion: set a maximum Pfa first, compute the CFAR threshold γ, then calculate the resulting Pd.

📐 Key Equations

  • CFAR Threshold (Eq.22): $\gamma = -\ln(P_{fa})$ → for $P_{fa}=10^{-6}$, $\gamma=13.82$
  • Probability of False Alarm (Eq.23): $P_{fa} = e^{-\gamma} = 10^{-6}$
  • Probability of Detection (Eq.24): $P_d = \frac{1}{2}\,\text{erfc}\!\left(\dfrac{\gamma - \text{SNR}_{\text{CI}}}{\sqrt{2\,\text{SNR}_{\text{CI}}}}\right) \approx 1.0$

🔢 MATLAB Code

Pfa_target   = 1e-6;
gamma_thresh = -log(Pfa_target);       % = 13.816
SNR_CI_linear = 10^(SNR_CI_dB/10);
Pd_approx = 0.5 * erfc( ...
    (gamma_thresh - SNR_CI_linear) / sqrt(2*SNR_CI_linear) );
Pd_approx = max(0, min(1, Pd_approx));

💡 Why It Matters

Pd vs SNR curve (Pfa = 10⁻⁶) — our operating point at 62 dB gives Pd ≈ 1.0

With SNR ~65 dB after integration the threshold of 13.82 is vastly exceeded, giving $P_d \approx 1.0$. In real systems with path loss and RCS fluctuation, this analysis identifies the minimum detectable SNR for a given Pd/Pfa requirement.

📊 Code Tracing Values

Target Pfa: 1×10⁻⁶

CFAR Threshold γ: 13.816

SNR_CI (linear): ~3.2×10⁶

Pd: ≈ 1.0000 (100% detection)

Block 13

Processing Latency Analysis

🔍 What It Does

Measures the execution time of each processing block to identify computational bottlenecks and assess whether real-time operation is feasible.

📐 Key Equations

  • Per-block Latency: $L_i = \text{toc}(t_{\text{start},i})$ — MATLAB timer per block
  • Total Latency (Eq.27): $L_{\text{total}} = \sum_{i=1}^{13} L_i$
  • Real-Time Constraint: $L_{\text{total}} \leq \text{PRI} = 35.6\ \mu\text{s}$ for real-time

🔢 MATLAB Code

t_total_start = tic;          % Start at top of script
t_blk6 = tic;
% ... matched filter ...
lat_blk6 = toc(t_blk6);
lat_total = toc(t_total_start);

fprintf('%-22s  %10s\n', 'Block', 'Time (ms)');
for i = 1:length(lat_blocks)
    fprintf('%-22s  %10.3f\n', block_names{i}, lat_blocks(i)*1e3);
end

💡 Why It Matters

Per-block latency — Matched Filter is the bottleneck at 46.3 ms (32.9%)

The Matched Filter (Block 6) is the heaviest block due to convolution over 128 pulses. In MATLAB simulation the total is ~5–20 ms on a modern CPU. Real-time hardware (FPGA/DSP) must meet the 35.6 μs PRI constraint — this analysis guides hardware acceleration decisions.

%% ================== PULSE-DOPPLER LFM RADAR SYSTEM ==================
%  Academic Project - Range, Velocity, SNR, Detection & Latency
%  All equations validated against Mahafza textbook
%  Extended: Pd, Pfa, SNR analysis, Ambiguity, Latency
% =====================================================================

%% ================== LATENCY TIMER START ==================
t_total_start = tic;  % Start total pipeline timer

%% ================== CONSTANTS ==================
% (Eq.1) Speed of light [1]
% c = 3 × 10^8 m/s
c = 3e8;

fs = 20e6;
Ts = 1/fs;

fc = 2.4e9;

% (Eq.2) Wavelength [1]
% λ = c / f_c
lambda = c / fc;

%% ================== TARGET ==================
target_range    = 500;   % m
target_velocity = 200;   % m/s
target_RCS      = 1.0;   % m² (assumed unit RCS for simulation)

%% ================== SIGNAL ==================
num_pulses     = 128;
pulse_len      = 512;
pulse_gap      = 200;
pulse_total_len = pulse_len + pulse_gap;

%% ================== LFM SIGNAL GENERATION ==================
t_blk1 = tic;

t_pulse = (0:pulse_len-1)/fs;
B = 5e6;

% (Eq.3) Pulse duration [2]
% T = N / f_s
T = pulse_len / fs;

% (Eq.4) Chirp rate [2]
% k = B / T
k = B / T;

% (Eq.5) LFM waveform [2]
% s(t) = exp(jπkt²)
pulse = exp(1j*pi*k*t_pulse.^2);

lat_blk1 = toc(t_blk1);   % Block 1 latency

%% ================== PULSE TRAIN FORMATION ==================
t_blk2 = tic;

tx = [];
for i = 1:num_pulses
    tx = [tx pulse zeros(1, pulse_gap)];
end

lat_blk2 = toc(t_blk2);   % Block 2 latency
fprintf('TX generated\n');

%% ================== CHANNEL (DELAY + DOPPLER) ==================
t_blk3 = tic;

% (Eq.6) Time delay [1]
% τ = 2R / c
tau = 2 * target_range / c;
delay_samples = round(tau * fs);
rx = [zeros(1, delay_samples) tx];

% (Eq.7) Doppler frequency [1]
% f_d = 2v / λ
fd = 2 * target_velocity / lambda;

t = (0:length(rx)-1) * Ts;

% (Eq.8) Received signal model [2]
% r(t) = s(t-τ) · exp(j2πf_d t)
rx = rx .* exp(1j*2*pi*fd*t);

lat_blk3 = toc(t_blk3);   % Block 3 latency
fprintf('Channel applied\n');

%% ================== AWGN NOISE ADDITION ==================
t_blk4 = tic;

% (Eq.9) AWGN model [4]
% n(t) ~ CN(0, σ²)
noise_power = 0.01;

% (Eq.19) Input SNR [3]
% SNR_in = Signal_power / Noise_power
signal_power_in = mean(abs(pulse).^2);
SNR_in_linear  = signal_power_in / noise_power;
SNR_in_dB      = 10 * log10(SNR_in_linear);

noise = sqrt(noise_power/2) * (randn(size(rx)) + 1j*randn(size(rx)));
rx = rx + noise;

lat_blk4 = toc(t_blk4);   % Block 4 latency
fprintf('Noise added  |  Input SNR = %.1f dB\n', SNR_in_dB);

%% ================== SLOW-TIME / FAST-TIME MATRIX ==================
t_blk5 = tic;

rx = rx(1:num_pulses*pulse_total_len);

% Reshape into slow-time / fast-time matrix [2]
rx_matrix = reshape(rx, pulse_total_len, num_pulses);

lat_blk5 = toc(t_blk5);   % Block 5 latency
fprintf('Matrix reshaped\n');

%% ================== MATCHED FILTER (PULSE COMPRESSION) ==================
t_blk6 = tic;

% (Eq.10) Matched filter definition [2]
% h(t) = s*(-t)
tx_mf = conj(fliplr(pulse));

range_matrix = zeros(pulse_total_len, num_pulses);

for i = 1:num_pulses
    rx_pulse = rx_matrix(:, i) .* hamming(pulse_total_len);

    % (Eq.11) Matched filtering (convolution) [2]
    % y(t) = r(t) * h(t)
    corr = conv(rx_pulse, tx_mf, 'full');
    corr = corr(pulse_len:end);
    corr = corr(1:pulse_total_len);
    range_matrix(:,i) = corr;
end

% (Eq.20) SNR after Matched Filter [2]
% SNR_MF = SNR_in × BT   (pulse compression gain)
BT         = B * T;                        % Time-bandwidth product = 256
G_MF_dB    = 10 * log10(BT);              % ~24.1 dB
SNR_MF_dB  = SNR_in_dB + G_MF_dB;

lat_blk6 = toc(t_blk6);   % Block 6 latency
fprintf('Matched filter done  |  SNR after MF = %.1f dB\n', SNR_MF_dB);

%% ================== RANGE ESTIMATION ==================
t_blk7 = tic;

% (Eq.12) Range from delay [1]
% R = (cτ)/2
[~, range_bin] = max(sum(abs(range_matrix), 2));

% (Eq.13) Discrete range axis [2]
range_axis = (0:pulse_total_len-1) * (c/(2*fs));
range_est  = range_axis(range_bin);

lat_blk7 = toc(t_blk7);   % Block 7 latency

%% ================== DOPPLER PROCESSING (SLOW-TIME FFT) ==================
t_blk8 = tic;

slow_time_signal = range_matrix(range_bin, :);

% Remove DC (clutter suppression) [2]
slow_time_signal = detrend(slow_time_signal);

% Windowing in slow-time [2]
slow_time_signal = slow_time_signal .* hamming(num_pulses).';

% (Eq.14) PRI and PRF [1]
PRI = pulse_total_len * Ts;
PRF = 1 / PRI;

N_fft = 1024;

% (Eq.15) Doppler FFT [2]
% S(f) = FFT{s[m]}
doppler_fft = fftshift(fft(slow_time_signal, N_fft));

% Frequency axis
f_axis = (-N_fft/2:N_fft/2-1) * (PRF/N_fft);

[~, idx_fd] = max(abs(doppler_fft));

% (Eq.21) SNR after Coherent Integration [2]
% SNR_CI = SNR_MF × M   (M = number of pulses)
G_CI_dB   = 10 * log10(num_pulses);       % ~21 dB
SNR_CI_dB = SNR_MF_dB + G_CI_dB;

lat_blk8 = toc(t_blk8);   % Block 8 latency

%% ================== QUADRATIC PEAK INTERPOLATION ==================
t_blk9 = tic;

% (Eq.16) Quadratic interpolation [2]
if idx_fd > 1 && idx_fd < N_fft
    alpha = abs(doppler_fft(idx_fd-1));
    beta  = abs(doppler_fft(idx_fd));
    gamma = abs(doppler_fft(idx_fd+1));
    delta = 0.5 * (alpha - gamma) / (alpha - 2*beta + gamma);
else
    delta = 0;
end

fd_est = f_axis(idx_fd) + delta*(PRF/N_fft);

% (Eq.17) Velocity estimation [1]
% v = (f_d · λ)/2
velocity_est = (fd_est * lambda) / 2;

lat_blk9 = toc(t_blk9);   % Block 9 latency
fprintf('Doppler estimated\n');

%% ================== RANGE-DOPPLER MAP ==================
t_blk10 = tic;

range_window  = hamming(pulse_total_len);
doppler_window = hamming(num_pulses);
range_matrix_win = range_matrix .* (range_window * doppler_window.');

% (Eq.18) 2D FFT (Range-Doppler transform) [2]
RD_map = fftshift(fft(range_matrix_win, N_fft, 2), 2);

RD_map_abs = abs(RD_map);
RD_map_abs = RD_map_abs / max(RD_map_abs(:));   % Normalize

velocity_axis = (f_axis * lambda) / 2;

lat_blk10 = toc(t_blk10);  % Block 10 latency
fprintf('RD map generated\n');

%% ================== SNR ANALYSIS ==================
fprintf('\n=========== SNR ANALYSIS ===========\n');

% (Eq.22) Power in dB [3]
% P(dB) = 20 log10(|x|)
fprintf('Input SNR              = %6.2f dB\n', SNR_in_dB);
fprintf('After Matched Filter   = %6.2f dB  (+%.1f dB, G = BT = %d)\n', ...
    SNR_MF_dB, G_MF_dB, round(BT));
fprintf('After Coherent Integr. = %6.2f dB  (+%.1f dB, G = M = %d)\n', ...
    SNR_CI_dB, G_CI_dB, num_pulses);
fprintf('Total Processing Gain  = %6.2f dB\n', G_MF_dB + G_CI_dB);

%% ================== DETECTION THEORY (Pd & Pfa) ==================
fprintf('\n=========== DETECTION THEORY ===========\n');

% (Eq.23) Probability of False Alarm [4]
% Pfa = exp(-γ)   for CFAR with exponential noise distribution
% γ = detection threshold
Pfa_target = 1e-6;                  % Desired Pfa = 10^-6

% CFAR threshold from Pfa [4]
% γ = -ln(Pfa)
gamma_thresh = -log(Pfa_target);    % Eq.23: γ = -ln(Pfa)

% (Eq.24) Probability of Detection (Neyman-Pearson) [4]
% Pd = Q_M(sqrt(2·SNR), sqrt(2·γ))
% Approximation using Albersheim's equation for CFAR detector:
% SNR_min(dB) = -5*log10(M) + [6.2 + 4.54/sqrt(M+0.44)] * log10(A + 0.12*A*B + 1.7*B)
% where A = ln(0.62/Pfa), B = ln(Pd/(1-Pd))
% Here we solve directly: given SNR_CI, compute Pd

% Use marcum Q approximation for non-coherent case
% For coherent integration: Pd ≈ Q(sqrt(2*SNR) - sqrt(2*gamma_thresh))
% Simplified Gaussian approximation:
SNR_CI_linear = 10^(SNR_CI_dB/10);
Pd_approx = 0.5 * erfc( (gamma_thresh - SNR_CI_linear) / sqrt(2*SNR_CI_linear) );
Pd_approx = max(0, min(1, Pd_approx));   % Clamp [0,1]

% Recompute actual Pfa from threshold for verification
Pfa_actual = exp(-gamma_thresh);

fprintf('Detection Threshold γ  = %.4f\n', gamma_thresh);
fprintf('Target Pfa             = %.2e\n', Pfa_target);
fprintf('Actual  Pfa            = %.2e\n', Pfa_actual);
fprintf('SNR after integration  = %.2f dB  (linear = %.1f)\n', SNR_CI_dB, SNR_CI_linear);
fprintf('Probability of Detect. = %.4f  (%.2f%%)\n', Pd_approx, Pd_approx*100);

% Unambiguous Range and Velocity
% (Eq.25) Unambiguous Range [1]
% R_unamb = c · PRI / 2
R_unamb = c * PRI / 2;

% (Eq.26) Unambiguous Velocity [1]
% v_unamb = λ · PRF / 4
v_unamb = lambda * PRF / 4;

fprintf('\nUnambiguous Range      = %.1f m\n', R_unamb);
fprintf('Unambiguous Velocity   = ±%.1f m/s\n', v_unamb);

%% ================== LATENCY ANALYSIS ==================
fprintf('\n=========== LATENCY ANALYSIS ===========\n');

% (Eq.27) Total Processing Latency [5]
% L_total = Σ L_i  for i = 1..N_blocks
lat_total = toc(t_total_start);

lat_blocks = [lat_blk1, lat_blk2, lat_blk3, lat_blk4, lat_blk5, ...
              lat_blk6, lat_blk7, lat_blk8, lat_blk9, lat_blk10];
block_names = {'LFM Generation', 'Pulse Train', 'Channel+Doppler', ...
               'Noise Addition', 'Matrix Reshape', 'Matched Filter', ...
               'Range Estim.', 'Doppler FFT', 'Interpolation', 'RD Map'};

fprintf('%-22s  %10s  %8s\n', 'Block', 'Time (ms)', '% Total');
fprintf('%s\n', repmat('-', 1, 46));
for i = 1:length(lat_blocks)
    pct = 100 * lat_blocks(i) / lat_total;
    fprintf('%-22s  %10.3f  %7.1f%%\n', block_names{i}, lat_blocks(i)*1e3, pct);
end
fprintf('%s\n', repmat('-', 1, 46));
fprintf('%-22s  %10.3f  %8s\n', 'TOTAL', lat_total*1e3, '100%');

%% ================== FINAL RESULTS ==================
fprintf('\n=========== FINAL RESULTS ===========\n');
fprintf('True Range         = %.2f m\n', target_range);
fprintf('Estimated Range    = %.2f m\n', range_est);
fprintf('Range Error        = %.2f m\n', abs(range_est - target_range));

fprintf('\nTrue Velocity      = %.2f m/s\n', target_velocity);
fprintf('Estimated Velocity = %.2f m/s\n', velocity_est);
fprintf('Velocity Error     = %.4f m/s\n', abs(velocity_est - target_velocity));

fprintf('\nPd  = %.4f (%.2f%%)\n', Pd_approx, Pd_approx*100);
fprintf('Pfa = %.2e\n', Pfa_actual);
fprintf('Total Latency = %.3f ms\n', lat_total*1e3);

%% ================== PLOT ==================

% (Eq.22) Power in dB
figure;
imagesc(range_axis, velocity_axis, 20*log10(RD_map_abs.'));
caxis([-40 0]);
xlabel('Range (m)');
ylabel('Velocity (m/s)');
title('Range-Doppler Map (ACADEMIC VERSION)');
colorbar;
axis xy;

fprintf('\nPlot done\n');

%% ================== END ==================
